
<h2>1.1. Основные аспекты современной командной разработки.</h2>

Системы для организации командной работы разделяются на следующие:

- Системы контроля версий (git, mercurial);
- Профессиональные чаты (rocket.chat, flowdock, slack);
- Таск - трекеры (Jira, Basecamp).
-  Хостинги для разработки ПО (GitHub, GitLab).

Всё это входит в понятие **инфраструктура разработки.**

<h2>1.2. Системы контроля версий, основные функции</h2>

Система контроля версий (системы управления версий, version control system)

**Преимущества git:**
- Более гибкое ветвление;
- Быстрая фиксация изменений;
- Возможность работать offline.

<h2>1.3.  Краткая история создания программного продукта или как системы контроля версий вписываются в процессы</h2>

**Коммит** - это действие по сохранению изменений в репозитории. 
**Ветки** позволяют изолировать изменения от двух разных специалистов. Каждый работает в своей ветке. 
**Слияние** - процесс сведения вместе изменений из нескольких веток. 
**Откат** позволяет управлять историей изменений в git как необходимо.
